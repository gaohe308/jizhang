小程序落地映射

下面这部分把原型视觉与微信原生小程序 + TypeScript + TDesign 组件方案对齐，便于下一步拆成真实页面和组件。

Design to Code

页面拆分建议

`pages/home/index`
首页总览、建房、最近房间、战绩摘要。
入口页
`pages/room/index`
房间成员、积分看板、流水广播、偏好设置。
核心页
`components/amount-keyboard`
自定义数字键盘、单发 / 群发模式切换、确认提交态。
弹层组件
`pages/settlement/index`
结算确认、轧差方案、最终支付指令。
结果页
`pages/profile/index`
个人战绩看板、历史记录、清空数据二次确认。
沉淀页

TDesign 组件建议

`t-navbar` + `t-tab-bar`
承担顶部导航与底部主导航，保证小程序原生操作习惯。
`t-card` + `t-cell` + `t-tag`
实现首页摘要卡片、规则提示、流水和战绩列表。
`t-popup` + 自定义数字键盘
点击头像后从底部弹出，保留高频输入的效率与沉浸感。
`t-dialog` + `t-action-sheet`
用于房间结算、清空历史、规则修改等二次确认场景。
`t-notice-bar` + `t-loading`
承接弱网、重连、提交中、同步成功等状态反馈。
原型重点：弱化噪音、强化账目层次、让“高频点击 + 实时同步 + 最终结算”三个关键任务在视觉上足够轻、足够稳、足够清楚。
