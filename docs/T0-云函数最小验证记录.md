# T0 云函数最小验证记录

## 验证目标

验证以下问题：

1. 当前 CloudBase 环境是否可以创建云函数
2. 云函数是否可以挂载到数据库所在 VPC / 子网
3. 云函数是否至少可以到达 CloudBase MySQL 的私网地址
4. Prisma 是否可以在云函数中按当前最小方式运行

## 本次新增验证函数

### 1. `t0-http-mysql-check`

类型：

- HTTP 云函数

用途：

- 提供 `/health`
- 提供 `/db-check`
- 用于验证 HTTP 云函数入口与 MySQL 连通性

目录：

- `cloudfunctions/t0-http-mysql-check`

### 2. `t0-db-probe`

类型：

- Event 云函数

用途：

- 直接执行 `SELECT 1 AS ok`
- 绕过网关因素，单独验证“云函数运行时 + VPC + MySQL”链路

目录：

- `cloudfunctions/t0-db-probe`

### 3. `t0-prisma-probe`

类型：

- Event 云函数

用途：

- 用最小 Prisma 调用验证当前环境下 Prisma 打包与运行可行性

目录：

- `cloudfunctions/t0-prisma-probe`

## 4. Prisma 最小验证已成功

`t0-prisma-probe` 已完成以下修正：

- 不再依赖默认的 `.prisma/client` 隐藏目录解析
- 改为显式引用 `./generated/client`
- 手动补齐 CloudBase 运行时所需的 `rhel-openssl-1.1.x` Query Engine
- 在函数启动时移除 `DATABASE_URL` 中的 `sslaccept` 参数
- 对 Prisma 查询结果做 JSON-safe 转换，避免 `BigInt` 序列化失败

最新调用结果：

- `ok: true`
- `countRows: [{ total: 0 }]`
- `sampleRows: []`

这说明：

- Prisma Client 已经可以在当前 CloudBase 云函数运行时成功启动
- Prisma 已经可以通过当前 VPC / 子网访问 CloudBase MySQL
- 当前数据库里 `users` 表暂无数据，因此 `sampleRows` 为空是正常现象

结论：

- **Prisma + MySQL 在当前云函数场景下已经完成最小验证**
- 后续可以把这套打包策略提炼到正式函数构建流程中
